1. What You Will Build

The session produces a working Fivetran ODI demo end-to-end. SQL Server (CDC via Fivetran) lands in an MDLS-managed Iceberg layer on S3 (cataloged by AWS Glue). dbt transforms that data through bronze, silver, and gold layers into Snowflake or Athena. A React frontend surfaces the results as a customer-facing demo application.

The full pipeline runs with real connectors, real infrastructure, and real data, not mocked output.

2. Setup — the easy path

Everything you need is in one public repo. Open Terminal and run these three lines:

# clone the repo, then run the setup script
git clone https://github.com/fivetran-jasonchletsos/odi-demo-builder.git
cd odi-demo-builder
./setup.sh

setup.sh auto-installs every Homebrew-based tool (Claude Code, git, GitHub CLI, 1Password CLI, Terraform, AWS CLI, Node 20, dbt), installs the demo skill in the exact layout Claude Code expects, then runs a preflight check and prints precisely what is left for you to do. It is safe to run more than once.

What it cannot do for you. A few steps need your own credentials and must be done by hand. The setup script lists each one with the exact command. They are: claude login, gh auth login, op signin, aws configure, your git name and email, and your Fivetran API key. See step 3.

Confirm you are ready

At any time, run the read-only preflight check. It changes nothing and reports the status of every prerequisite, with a copy-and-paste fix for anything missing:

./preflight.sh

Success looks like: the final line reads All set. You are ready for the session. If anything is marked to fix, run the command it shows you, then run ./preflight.sh again.

Make the skill load

Claude Code discovers skills at startup, so after setup, quit and reopen Claude Code. Then confirm the skill is live by running:

./preflight.sh

The skill line will read PASS, and typing /odi-demo-builder inside a Claude Code session will resolve.

3. The six credential steps

Run each one. The setup script prints these too, so you can copy them from your terminal.

a. Claude Code login

claude login

Follow the browser prompt. Success: your Anthropic email shows when you run claude whoami.

b. GitHub CLI

gh auth login

Choose GitHub.com, HTTPS, and authenticate in the browser. Success: gh auth status says you are logged in.

c. 1Password CLI

op signin

Enter your 1Password account URL and credentials when prompted. Success: op whoami prints your account email.

d. AWS CLI

aws configure

Provide your Access Key ID, Secret Access Key, default region (us-east-1 or your team standard), and output format json. Success: aws sts get-caller-identity returns your account.

e. Git identity

git config --global user.name "Your Name"
git config --global user.email "you@fivetran.com"

Success: git config --global user.email prints your email.

f. Fivetran API key

Get it from fivetran.com, click your avatar (top right), then API key. Copy the key and secret. Add them to ~/.zshrc:

export FIVETRAN_API_KEY="your_key_here"
export FIVETRAN_API_SECRET="your_secret_here"

Then reload your shell:

source ~/.zshrc

Success: the Fivetran line in ./preflight.sh reads PASS.

4. If something goes wrong

Run ./preflight.sh. Every failing item prints the exact command to fix it. If you are still stuck, copy the full preflight output and paste it in the shared Slack channel before the session. Do not wait until the session starts to sort out a broken tool.

5. What to Expect in the Session

The session runs in four phases. Each builds directly on the last.

  • Phase 1: Gather. Claude Code interviews you about the customer scenario, connector topology, and destination. It produces a structured spec that drives the rest of the session.
  • Phase 2: Provision. Terraform modules are generated and applied, creating the S3 bucket, Glue catalog database, IAM roles, and the Fivetran connector via the API. You review the plan before anything is applied.
  • Phase 3: Build. dbt models (bronze, silver, gold) and the React frontend are scaffolded and wired to your provisioned infrastructure. You validate model grain and business logic at each layer.
  • Phase 4: Run. The end-to-end pipeline executes: Fivetran syncs, dbt transforms, the React app loads. You confirm the demo is presentable and tag a release commit.

6. A Note for Andy

The SQL Server to MDLS leg of this pipeline is the HVR replacement pitch. You already know the architecture better than Claude Code does, and the builder knows this and will defer to you at a specific decision point.

During the Gather phase, Claude Code will ask you to choose between CDC (log-based) and XACT (transaction-based) replication for the SQL Server connector. It surfaces both options but waits for your call. Make the choice you would make for a data-residency-sensitive enterprise customer, because that is the next thing it shows you: the Hybrid deployment option appears immediately after, with the agent explaining why it is the right fit for customers who cannot send CDC log data outside their firewall.

This is intentional. The demo is structured so that your expertise drives the narrative, not a preset script. Pay attention to how the builder surfaces the Hybrid option. That framing is what you would use with a CDO who is replacing HVR and nervous about cloud egress.
